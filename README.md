# OpenAPITemplate.jl

A [PkgTemplates.jl](https://github.com/JuliaCI/PkgTemplates.jl) plugin that
scaffolds Julia REST/JSON API wrapper packages.

The generator produces a two-layer architecture:

- `src/api/` — generated by [OpenAPI Generator](https://openapi-generator.tech/)
  (`julia-client`), committed as plain Julia, never re-run at end-user runtime.
- `src/client/` — hand-written ergonomic overlay providing a `Client` struct,
  middleware, auth strategies, pagination iterators, and pretty printing.

See [`IMPLEMENTATION_PLAN.md`](IMPLEMENTATION_PLAN.md) for the full design and
phased rollout.

## Quick start

```julia
using PkgTemplates, OpenAPITemplate

t = Template(;
    user = "your-username",
    plugins = APIWrapper(;
        spec_url = "https://petstore3.swagger.io/api/v3/openapi.json",
    ),
)
t("PetstoreClient")
```

## Status

- **Phase 1 — MVP plugin skeleton** ✅
- **Phase 2 — OpenAPI codegen integration** ✅ (drives `julia-client` via the
  `npx` wrapper at scaffold time; populates `src/api/`, writes
  `gen/regenerate.jl` for later refreshes, and adds `Dates` + `TimeZones` deps
  the generated module needs.)

Ergonomic middleware, reliability primitives, testing infrastructure, and
docs land in Phases 3–6.

## Maintainer prerequisites

Scaffolding **with** a spec needs Java 11+ and Node 18+ on PATH. End users of
the generated package never need either — the committed `src/api/` is plain
Julia source.
